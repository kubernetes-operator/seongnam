# CLI Interface 에이전트

## 핵심 역할

Rich 라이브러리 기반 CLI 도구를 구현하여 터미널에서 모니터링 데이터를 표(Table) 형태로 시각화한다. API 서버에서 데이터를 가져와 색상·아이콘·진행 바를 활용한 직관적인 터미널 UI를 제공한다.

## CLI 명령 구조

```
k8s-monitor
├── status                    # 전체 클러스터 현황 (기본 명령)
│   ├── --cluster <name>      # 특정 클러스터만
│   └── --watch               # 30초 자동 갱신
├── nodes                     # 노드별 상세 현황
│   ├── --cluster <name>
│   └── --sort <metric>       # cpu|memory|disk|load 정렬
├── os                        # OS 영역 메트릭
│   ├── --cluster <name>
│   └── --node <name>
├── k8s                       # Kubernetes 영역 메트릭
│   ├── --cluster <name>
│   └── --node <name>
├── report                    # 리포트 조회/생성
│   ├── list                  # 리포트 목록
│   ├── show <report_id>      # 리포트 내용
│   └── generate              # 새 리포트 생성
│       ├── --type daily|weekly|monthly|yearly
│       └── --cluster <name>
├── events                    # 위기 이벤트 목록
│   ├── --severity warning|critical
│   └── --unresolved
├── predict                   # 예측 결과
│   └── --cluster <name>
└── top                       # 고사용량 Top N
    ├── --metric cpu|memory|disk
    └── --limit 10
```

## 구현 원칙

1. **Rich 라이브러리 사용**: `rich.table`, `rich.console`, `rich.progress` 활용
2. **색상 기반 상태 표시**: 정상(green), 경고(yellow), 위험(red), 불명(white)
3. **최대 대비 비율 강조**: 임계값 초과 시 `[값]🔴` 형태로 강조 표시
4. **API 호출**: API 서버 REST API를 `httpx` 라이브러리로 호출
5. **설정 파일**: `~/.k8s-monitor/config.yaml`에 API 서버 URL, API Key 저장
6. **오프라인 모드**: API 접근 불가 시 마지막 캐시 데이터 표시 (캐시 시간 명시)

## 설정 파일 형식

```yaml
# ~/.k8s-monitor/config.yaml
api_url: http://k8s-monitor.internal
api_key: ${K8S_MONITOR_API_KEY}
default_cluster: prod-cluster-01
output:
  max_rows: 50
  date_format: "%Y-%m-%d %H:%M UTC"
  timezone: "Asia/Seoul"
```

## 에러 핸들링

- API 연결 실패: 빨간 오류 박스로 표시, 재연결 안내
- 빈 데이터: "데이터 없음" 메시지와 함께 수집 시작 명령 안내
- 인증 실패: API Key 설정 방법 안내

## 협업

- **api-server**: HTTP API 호출로 데이터 수신

## 팀 통신 프로토콜

수신: 사용자 CLI 입력
발신: api-server → HTTP 요청 (직접 API 호출, 팀 메시지 사용 안 함)
