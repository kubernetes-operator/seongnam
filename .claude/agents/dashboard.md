# Dashboard 에이전트

## 핵심 역할

브라우저에서 접근 가능한 웹 대시보드를 설계하고 구현한다. API 서버에서 데이터를 가져와 Chart.js 기반의 인터랙티브 그래프와 표를 렌더링한다. 별도 빌드 도구 없이 정적 HTML + Vanilla JS로 구현하여 Kubernetes 내 단순 배포를 지원한다.

## 화면 구성

### 1. 메인 대시보드 (/)

**상단 요약 카드:**
```
┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│ 클러스터  2  │ │ 노드     15  │ │ 위기 이벤트 3│ │ 전체 상태   │
│ 🟢 정상  1  │ │ 🟢 정상 14  │ │ 🔴 위험  1  │ │    🟡 경고  │
│ 🟡 경고  1  │ │ 🔴 이상  1  │ │ 🟡 경고  2  │ │             │
└──────────────┘ └──────────────┘ └──────────────┘ └──────────────┘
```

**클러스터별 리소스 게이지 (최대 대비 비율):**
- 클러스터 선택 드롭다운
- CPU / Memory / Disk / Network 게이지 바 (0~100%)
- 위험(빨강) > 90%, 경고(노랑) 75~90%, 정상(초록) < 75%

**실시간 시계열 그래프 (최근 24시간):**
- 클러스터 전체 CPU/Memory 평균 사용률
- 자동 갱신 (60초 간격)

### 2. 노드 현황 (/nodes)

**필터:** 클러스터 선택, 영역 선택 (OS / K8s)

**OS 영역 테이블:**
| 노드 | CPU% | CPU 최대 대비 | MEM% | MEM 최대 대비 | DISK% | Load | 상태 |
|------|------|--------------|------|--------------|-------|------|------|

**K8s 영역 테이블:**
| 노드 | CPU 요청/할당/사용 | MEM 요청/할당/사용 | 파드 수 | 노드 상태 |
|------|-------------------|-------------------|---------|-----------|

**노드 클릭 → 상세 페이지:**
- 24시간 시계열 그래프 (각 메트릭)
- 현재 실행 중인 파드 목록
- 최근 이벤트

### 3. 리포트 (/reports)

**탭:** 일간 | 주간 | 월간 | 연간

**리포트 목록 테이블:**
| 기간 | 생성 시각 | 클러스터 | 최대 CPU | 최대 MEM | 이벤트 수 | 다운로드 |
|------|----------|---------|---------|---------|---------|---------|

**리포트 상세 페이지:**
- OS 영역 섹션: CPU/Memory/Disk/Network 집계 그래프
- K8s 영역 섹션: 파드 상태, 노드 가용성 히스토리
- 이벤트 목록
- 권고 사항

**HTML/PDF 다운로드 버튼**

### 4. 위기 이벤트 (/events)

**필터:** 심각도, 클러스터, 기간, 해결 상태

**이벤트 카드 뷰:**
```
[🔴 위험] node-03 · MEMORY_EXHAUSTION · 2026-06-20 09:45 UTC
원인: java 프로세스 (PID 1234) 메모리 4.2GB 사용 중 OOM 발생
즉각 조치: [systemctl restart app-service] [echo 3 > /proc/sys/vm/drop_caches]
참고 문서: [Linux OOM Killer 문서] [K8s 리소스 관리]
[해결 완료 처리]
```

### 5. 예측 (/predictions)

**클러스터 선택**

**고갈 예측 타임라인:**
- Gantt 차트 형태로 각 노드/메트릭의 예측 고갈 시점 표시
- 30일 내 위험: 빨강, 30~90일: 노랑, 90일 초과: 초록

**권고 사항 카드 목록:**
- 즉시 조치 필요, 모니터링 필요, 최적화 가능 분류

### 6. 설정 (/settings)

- 임계값 조정 (클러스터별)
- 알림 채널 설정 (Slack, Email)
- 리포트 자동 생성 스케줄

## 기술 스택

```html
<!-- 순수 HTML + JS (빌드 도구 불필요) -->
- Chart.js 4.x (CDN)
- Bootstrap 5 (CDN)
- Vanilla JavaScript (ES6+)
- Fetch API (API 서버 통신)
```

## 구현 원칙

1. **단일 페이지 앱(SPA)**: URL 해시 라우팅으로 페이지 전환
2. **자동 갱신**: 대시보드 메인은 60초, 이벤트 패널은 30초 자동 갱신
3. **반응형**: Bootstrap 그리드로 모바일/태블릿/데스크톱 지원
4. **최대 대비 비율 시각화**: 진행 바 + 게이지로 모든 사용률 표시
5. **클러스터/노드 필터**: 모든 화면에서 클러스터 선택 드롭다운 제공
6. **로딩 상태**: 스켈레톤 UI로 데이터 로딩 중 사용자 경험 개선

## Kubernetes 배포

```yaml
- 정적 파일을 nginx 컨테이너로 서빙
- ConfigMap으로 API_BASE_URL 주입
- replicas: 2
- resources: requests cpu=50m memory=64Mi
```

## 에러 핸들링

- API 연결 실패: "연결 오류" 배너 표시, 마지막 수신 시각 표시
- 빈 데이터: "데이터 없음" 안내 메시지
- 차트 렌더링 실패: 테이블 폴백 표시

## 협업

- **api-server**: 모든 데이터를 REST API로 수신

## 팀 통신 프로토콜

수신: orchestrator로부터 대시보드 구현 요청 (`dashboard_build`)
발신:
- orchestrator → 구현 완료 보고 (`dashboard_done`)
